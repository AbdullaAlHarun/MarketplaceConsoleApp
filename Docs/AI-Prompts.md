# AI Prompts Documentation

During development, AI assistance  was used as a support tool during development to get guidance and improve the structure of the application.


## Prompt 1 – Project Structure and OOP Design

I am building a console-based marketplace application in C#.
How should I structure the project using Models, Services, UI, and Helpers?
Can you explain how separation of concerns works in this context and why it is important for maintainability?

This helped clarify how to organize the project and separate responsibilities between layers.

---

## Prompt 2 – Domain Modeling and Relationships

In a marketplace system, how should I model the relationship between users, listings, transactions, and reviews?
What is the benefit of introducing a Transaction class instead of only updating listing status?"

This helped in designing the core domain models and understanding how data flows through the system.

---

## Prompt 3 – Business Logic and Rules

What is a clean way to implement business rules such as preventing users from buying their own listings, marking items as sold, and allowing reviews only after a purchase?
How should this logic be structured inside service classes?"

This guided the implementation of service-layer logic and ensured rules were handled consistently.

---

## Prompt 4 – LINQ and Search Functionality

How can LINQ be used in a console application to implement search and filtering of listings by keyword and category?
Can you explain the concept of chaining queries and how it improves readability?

This supported the implementation of search and filtering features.

---



## Notes

AI was used mainly for conceptual understanding, design guidance, and improving code organization.

The prompts above represent the main types of questions asked during development. Similar variations were used when refining features, debugging issues, and improving structure.

All code was written, reviewed, and tested manually before submission.
